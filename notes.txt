Node.js is a JavaScript runtime environment that lets you run JavaScript outside the browser.
Normally, JavaScript runs only inside browsers like Chrome.
With Node.js, you can run JavaScript on your computer/server to build:

Backend APIs
Web servers
Real-time chat apps
Authentication systems
File handling
Databases connections
Command-line tools

🔥 Video-6 : How Node js works - Architecture  of Node js
It starts with a client/user who request to the server(currently its Node js)
now we'll see how node js will handle this request
First step is event queue
request will be queuued inside the event queue
what event the event present inside the event queue will go to event loop
event loop is nothing but it'll monitor our event queue
It folllows FIFO
when we pick the req from the queue it can be of 2 type 1.blocking operation(sync op) 2.non-blocking operation(non-sync op)
event checks the type os req is its non-blocking it'll excute immedatly and user will get the output
if its blocking req it'll go near thread pool
now what is thread pool? its a pool containing threads - its a worker who'll work for you
this thread will helps you in blocking operations
each blocking op has a thread 
once its done it'll give you the desired data

Default thread pool size is 4
we can increase it depending on Cpu is i have 8core cpu i can have max 8 threads
